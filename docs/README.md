## ğŸ¤ Contributing to Documentation

### Adding New Documentation

1. Create markdown file in appropriate directory
2. Follow documentation structure
3. Add to DOCUMENTATION.md index
4. Update this README

### Style Guide

- Use **clear headings** (## for main sections)
- Include **code examples** with syntax highlighting
- Add **links** to related documentation
- Use **tables** for comparisons
- Include **diagrams** where helpful (ASCII or Mermaid)
- Provide **examples** for complex topics

### Documentation Review

Before submitting documentation:

1. **Proofread** for typos and grammar
2. **Test all commands** and examples
3. **Verify all links** work
4. **Check consistency** with other docs
5. **Update index** (DOCUMENTATION.md)
